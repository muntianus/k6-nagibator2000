## 🛡️ Гайд по безопасности при работе с Vast.ai

Этот документ содержит ключевые принципы и правила для безопасного использования облачной платформы Vast.ai.

### Модель угроз

1.  **Риски для Арендатора (вас):**
    *   **Недобросовестный хост:** Владелец физической машины теоретически имеет доступ к оперативной памяти и диску. Он может попытаться получить доступ к вашим данным или секретам.
    *   **Сетевой перехват:** Передача данных по незащищенным протоколам может быть перехвачена.

2.  **Риски для Хоста (владельца GPU):**
    *   **Недобросовестный арендатор:** Может использовать мощности для незаконной деятельности.
    *   **"Побег из контейнера":** Попытка получить доступ к основной ОС хоста из Docker-контейнера.

### Меры защиты со стороны Vast.ai

*   **Контейнерная изоляция:** Ваш код всегда выполняется внутри **Docker-контейнера**, что является основным барьером безопасности.
*   **SSH-туннелирование:** Стандартные подключения (`ssh`, `scp`) защищены шифрованием.
*   **Рейтинговая система:** Показатель `reliability` (надежность) помогает выбирать проверенных провайдеров.

### Ваша зона ответственности (Обязательные правила)

**Главное правило: не обрабатывайте на Vast.ai критически важные, личные или коммерческие данные в незашифрованном виде.**

1.  **Шифруйте данные:** Всегда шифруйте датасеты и важные артефакты перед загрузкой на инстанс.
2.  **Используйте переменные окружения:** Для передачи API-ключей, токенов и паролей используйте флаг `--env`. **Никогда не вшивайте секреты в код.**
3.  **Используйте приватные Git-репозитории:** Код и модели загружайте из приватных репозиториев с помощью одноразовых deploy-ключей.
4.  **Выбирайте проверенных хостов:** Отдавайте предпочтение машинам с `reliability > 0.98` и большим количеством успешных аренд.
5.  **Считайте инстанс временным:** Относитесь к каждому инстансу как к временному и потенциально небезопасному окружению. Сохраняйте результаты работы на внешнее хранилище сразу после их получения.

### Итог

Vast.ai — это экономически эффективное, но менее защищенное решение по сравнению с AWS/GCP/Azure. Платформа идеально подходит для задач, не связанных с обработкой конфиденциальных данных.
